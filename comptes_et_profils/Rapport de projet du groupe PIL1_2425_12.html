2. Fonctionnement interne du groupe

Pour ce projet , notre groupe s'est partagé le travail en 6 parties , de sorte à ce que chacun puisse avoir sa part de travail . Les travaux ont donc officiellement commencé après l'établissement du plan de travail et le partage des tâches . Le travail était divisé en 2 volets : le front-end , et le back-end . Pendant que le front-end s'occupait de l'interface des pages , le back-end recuillait au fur et à mesure les résultats pour établir les relations en base de données , le recueil des informations utilisateurs , la maintenance de l'application due aux diverses modifications . En fin de projet , les responsables back-end se sont concentrés sur l'établissement de l'algorithme de matching .

Les travaux finaux (liens entre templates , établissement des styles , backend) ont dû être menés par les chargés du backend en raison de la divergence objective de certains résultats de membres du groupe , et des contraintes temporelles sur le projet .

3. Description de la conception

Structure du projet

Le projet s'est structuré en 3 blocks de codes principaux : les templates , les fichiers python (avec les packages primordiaux pour le fonctionnement : django , folium , geopy , mysqlclient) , et les fichiers statics . Ainsi , les fichiers principaux tels que ceux de création de compte ou du matching disposent de bases modulaires , favorisant la maintenance et les améliorations .

Base de données

Nous avons utilisé une base de données MySQL établie avec les classes éditées en Django , pour la créaton des tables. Voici les entités principales de notre base de données :

  • CustomUser
  • Message
  • Discussion
  • Profile
  • Trajet
  • Match

4. Instructions de déploiement

  • Cloner le dépôt GitHub
  • Installer les packages indispensables : python install django ; pip install geopy ; pip install folium ; pip install mysqlclient
  • Configurer la base MySQL dans settings.py
  • Lancer les migrations : pip manage.py makemigrations ; python manage.py migrate
  • Lancer le serveur : python manage.py runserver

5. Manuel d’utilisation

Pour utiliser la plateforme de transport Logisticar, l’utilisateur commence par créer un compte en se rendant sur la page d’inscription. Il doit remplir ses informations personnelles (nom, prénom, téléphone, email, mot de passe) et choisir s’il est conducteur ou passager. Si c’est un conducteur, il devra ensuite renseigner les informations sur son véhicule (marque, modèle, nombre de places). Une fois inscrit, il peut se connecter avec son email et son mot de passe, puis accéder à son profil, qu’il peut modifier à tout moment (horaires habituels, point de départ, etc). Le conducteur peut alors publier un trajet en précisant les lieux, horaires et nombre de places, tandis que le passager peut rechercher un trajet correspondant à ses besoins. Si un trajet l’intéresse, il peut envoyer un message privé au conducteur via la messagerie intégrée. Par ailleurs, un système de matching automatique lui proposera aussi des trajets compatibles avec ses préférences. En cas d’oubli de mot de passe, l’utilisateur peut le réinitialiser grâce à un lien envoyé par email. Enfin, une carte interactive permet de visualiser les trajets disponibles et les localisations. Tout est pensé pour que conducteurs et passagers puissent facilement se connecter, organiser leurs déplacements et communiquer de manière fluide sur la plateforme.
